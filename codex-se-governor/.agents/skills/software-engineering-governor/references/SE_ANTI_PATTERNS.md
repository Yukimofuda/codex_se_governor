# Software Engineering Anti-patterns

- Coding before reading the existing system.
- Treating prompt text as a complete requirement.
- Designing around current sample data only.
- Broad rewrite mixed with feature work.
- Adding dependencies without need, risk, and alternatives.
- Hiding behavior change inside refactoring.
- Using inheritance only for reuse.
- Using patterns to look sophisticated.
- Creating Singleton for convenience.
- Testing only the happy path.
- Treating a passing build as proof of correctness.
- Ignoring security because the change is small.
- Logging sensitive data.
- Letting AI decide core business rules.
- Accepting AI output without source reading, tests, and review.
