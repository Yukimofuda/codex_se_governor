# Software Engineering Checklist

- [ ] I read `AGENTS.md`.
- [ ] I read relevant code and tests.
- [ ] I identified stakeholders or stated why not applicable.
- [ ] I separated functional requirements, NFRs, constraints, and assumptions.
- [ ] I wrote or confirmed user story and acceptance criteria.
- [ ] I analyzed domain entities, boundaries, controls, relationships, data flow, and failure modes.
- [ ] I designed module boundaries, interfaces, data ownership, and quality trade-offs.
- [ ] I reviewed risk probability, impact, mitigation, and contingency.
- [ ] I kept implementation minimal and reversible.
- [ ] I avoided sample-specific logic.
- [ ] I reviewed SOLID and pattern pressure where structural design changed.
- [ ] I added or updated tests, or documented test gaps.
- [ ] I reviewed security, privacy, dependencies, error handling, logging, and deployment.
- [ ] I updated documentation if behavior, API, setup, or operations changed.
- [ ] I ran relevant commands and recorded results.
- [ ] I prepared rollback and memory update suggestions.
