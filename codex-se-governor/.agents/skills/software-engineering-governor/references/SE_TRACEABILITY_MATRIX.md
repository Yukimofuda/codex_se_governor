# Skill Traceability Matrix

This Skill is backed by `docs/software-engineering/18_TRACEABILITY_MATRIX.md`.

Minimum trace for every non-trivial task:

| Lifecycle Area | Required Evidence |
|---|---|
| Requirements | Requirement IDs, stakeholders, NFRs, constraints, conflicts |
| User Story | Role, goal, benefit, priority, estimate, acceptance criteria |
| Analysis | Domain concepts, EBC objects, relationships, failure modes |
| Design | Boundaries, interfaces, data model, SOLID, pattern justification |
| Testing | Test IDs mapped to requirement IDs and coverage intent |
| Security | Threats, trust boundaries, auth, input, secrets, dependencies |
| Risk | Probability, impact, mitigation, contingency, owner |
| Completion | Commands, docs, rollback, remaining risks, memory suggestions |
