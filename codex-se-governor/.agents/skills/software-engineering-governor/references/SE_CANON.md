# Software Engineering Canon

1. Software is programs, data, documentation, configuration, tests, and operational knowledge.
2. Software engineering is systematic, disciplined, measurable production, operation, and maintenance of software.
3. Quality is the base concern; process, methods, and tools exist to serve quality.
4. Requirement misunderstanding contaminates analysis, design, implementation, and tests.
5. Agile means small iterations, frequent feedback, working software, collaboration, and necessary documentation.
6. User stories need role, goal, benefit, priority, estimate, and acceptance criteria.
7. Analysis identifies problem domain concepts, entities, boundaries, controls, relationships, and failure modes.
8. Design protects maintainability through abstraction, encapsulation, low coupling, high cohesion, modularity, and justified refactoring.
9. Implementation follows local style, meaningful names, no unexplained policy numbers, reduced duplication, focused responsibilities, and reviewable patches.
10. Testing verifies requirements, reveals defects, covers boundaries, failures, regression, and security paths.
11. Architecture is driven by quality attributes and context.
12. Project management makes tasks, milestones, estimates, risks, resources, and communication visible.
13. Ethics and AI require fairness, bias awareness, transparency, accountability, privacy, and IP care.
14. Risk and quality management run throughout the project.
15. Secure development includes threat awareness, validation, authorization, secrets handling, dependency review, error handling, and deployment checks.
16. SOLID is a design review tool, not formalism.
17. Design patterns are justified responses to real design pressure.
18. AI-assisted development needs guardrails: tests, review, security scanning, human oversight, and measurable improvement.
