# Release Checklist

## Build

- [ ] Build command succeeds.

## Test

- [ ] Automated tests pass.
- [ ] Manual acceptance checks complete if required.

## Security

- [ ] Security review complete.
- [ ] Secrets and configuration checked.

## Documentation

- [ ] README/API/user docs updated.

## Migration

- [ ] Data/config migration documented.

## Rollback

- [ ] Rollback command or procedure documented.

## Monitoring

- [ ] Logs, metrics, alerts, or manual checks defined.

## Known Risks

- [ ] Remaining risks accepted by owner.
